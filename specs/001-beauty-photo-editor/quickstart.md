# Quickstart & Validation Guide: MyPix

**Branch**: `001-beauty-photo-editor` | **Plan**: [plan.md](./plan.md)

How to run the app and prove each phase works end-to-end. Implementation detail lives in [contracts/](./contracts/) and [data-model.md](./data-model.md); tasks live in tasks.md (generated by `/speckit-tasks`).

## Prerequisites

- Node.js 20+ and npm (or pnpm)
- Desktop browser for development; **iPhone 17 Pro (Safari)** for target-device validation
- Test photos: a large JPEG (~48 MP), a HEIC portrait, a full-body shot, a multi-face group shot, a no-face landscape

## Setup & run

```bash
npm install
npm run dev            # http://localhost:3000 — note: SW/offline behavior needs a production build
npm run build          # static export (output: 'export')
npm run start:static   # serve the export locally with COOP/COEP headers (mirrors vercel.json)
npm test               # Vitest unit suites (edit-state, preset codec, warp math, i18n parity)
npm run test:e2e       # Playwright flows (WebKit + mobile viewport)
```

Deploy: push to the connected Vercel project (free tier). `vercel.json` must ship the COOP/COEP headers (research R13) — verify `crossOriginIsolated === true` in the deployed console.

## Validation scenarios

### V1 — Import → full-res export backbone (Story 1.1, FR-116/117/118)
1. Open the app, add the 48 MP JPEG → displays at correct orientation.
2. Add the HEIC photo → displays without error (native decode or heic2any fallback).
3. Export the 48 MP photo as PNG and as JPEG → downloaded files have **exactly the original pixel dimensions** (check with any image inspector); app never blanks or reloads.
4. Tap Share → system share sheet opens with the image file; cancel → app offers plain save.

### V2 — Adjustments, curves, filters (Stories 1.2/1.4/1.5)
1. Scrub each of the 12 global adjustments → preview updates live while dragging (SC-009).
2. Bend the red curve only → only red response changes. Desaturate only the red band in the mixer → other colors untouched.
3. Apply a filter, tap again → intensity slider 0–100% blends proportionally. Stack vignette + grain → both survive export.
4. Reset any control → image returns exactly to prior state. Export → matches preview (compare side-by-side).

### V3 — Compare, undo, drafts (Stories 1.7/1.8)
1. Make ≥ 5 edits; drag the before/after divider; press-and-hold shows the original, release restores.
2. Undo ×3, redo ×3 → state steps correctly. Reopen an earlier adjustment, change it → later edits intact.
3. Mid-edit, reload the page → resume card appears; re-pick the same photo → edit restored with working undo. Re-pick a *different* photo → clear mismatch message, no silent wrong-apply.

### V4 — Bilingual & PWA (Stories 1.9/1.10, FR-004–008)
1. Toggle EN↔VI mid-edit → every visible string switches, edit state untouched; tone-mark-heavy strings (ế ộ ữ ẫ ợ ỳ) render unclipped. Restart → language remembered.
2. Run the i18n parity unit test → `en`/`vi` key sets identical, no empty values.
3. On the iPhone: follow in-app guidance → Add to Home Screen → launches standalone with correct icon/name.
4. Airplane mode → launch → shell loads; complete import→edit→export fully offline (SC-013, SC-015).

### V5 — Face intelligence (Stories 2.1–2.4)
1. Open face tools on a portrait → detection completes; adjust jaw/nose/eyes → only the targeted feature changes, background stable (SC-005).
2. Skin smoothing at moderate strength → export at full res → pores/texture still visible (SC-004); strength ramps smoothly.
3. Teeth whitening / eye brightening / under-eye reduction → each confined to its region.
4. Auto-beautify on a portrait → balanced result in ≤ ~3 s (SC-010), each component individually adjustable afterward.
5. No-face landscape → face tools clearly unavailable with manual-warp suggestion; multi-face photo → face picker appears.

### V6 — Body, warp, background (Stories 2.5–2.8)
1. Full-body photo → waist/leg/arm reshape natural; straight background lines acceptably intact.
2. Manual warp: push/pull with falloff; freeze an area → it stays fixed; loupe appears offset from the finger; reconstruct eases the warp back.
3. Background blur: subject sharp, edge clean at moderate strength, strength scales smoothly.
4. Background removal → solid color / grayscale replacement; edge-refine improves hair boundary; export transparent PNG → transparency preserved.

### V7 — Creative layer (Stories 3.1–3.8)
1. Makeup: apply lipstick/blush/eyeshadow/liner/brow → each tracks the face, individually removable/adjustable.
2. Vietnamese text overlay with stacked tone marks → correct on canvas **and** in full-res export.
3. Stickers place/transform/layer; drop a new file + index entry into `public/stickers/` → appears without code change.
4. Frames, clone/heal repair, blend modes, collage, doodle → each composes and exports at full resolution (FR-309).

### V8 — Target-device certification (Phase 4, run the `tests/device/` matrix)
On the installed PWA, iPhone 17 Pro: repeated 48 MP exports + face/body/segmentation across a long session → no crash/reload (SC-011/012); P3 photo preview vs export → no color shift (SC-003); full bilingual walk-through → no missing/clipped strings (SC-006–008); install → offline edit/export → online update → draft survives (SC-013, FR-404); one-handed reachability + safe areas (SC-019).

## Success gate

The feature is done when every scenario above passes **on the target device**, all unit/E2E suites are green, and the Constitution Check in plan.md still holds post-implementation.
