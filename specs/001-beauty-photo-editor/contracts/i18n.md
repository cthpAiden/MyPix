# Contract: i18n — Message Catalogs & Locale

**Scope**: `src/i18n/` — next-intl configuration and the `en`/`vi` message catalogs. Single source of truth for every user-facing string (Constitution IV).

## Structure

```text
src/i18n/
├── request.ts             # next-intl request config ([locale] segment: 'en' | 'vi')
├── routing.ts             # locales: ['en','vi'], defaultLocale: 'en', localePrefix
└── messages/
    ├── en.json
    └── vi.json
```

- Namespaced keys mirror the module tree: `common.*`, `import.*`, `export.*`, `tools.adjust.*`, `tools.face.*`, `errors.*`, `gift.*`, etc. Each tool module owns its `tools.<id>.*` namespace but strings live only in these two catalogs.
- ICU message format for interpolation/plurals; typed keys via next-intl's TypeScript augmentation so a missing/typo'd key is a compile error.

## Hard rules

1. **No literal user-facing strings in components** — everything through `useTranslations()`. (Lint rule: flag JSX text nodes and title/aria attributes outside `i18n/messages`.)
2. **Catalog parity is CI-enforced**: a unit test asserts `keys(en) === keys(vi)` deep-equal and that no value is empty or equal to its key. This makes FR-401 continuously true instead of a Phase 4 scramble.
3. **Vietnamese is written for a person, not translated literally** — warm second-person register per the spec's gift-voice direction; reviewed by a human in Phase 4 (FR-401 "natural phrasing").
4. **Layout**: no fixed-width labels; containers size to the longer language (vi ≈ +20–30%); line-height generous enough for stacked tone marks (spec bilingual layout rules, SC-008 no clipping).
5. **Language attribute**: rendered strings carry the active `lang` (`en`/`vi`) for assistive tech.

## Locale switching

- Visible persistent toggle (EN/VI) in the main interface (FR-004).
- Switch = client-side locale change via next-intl navigation **preserving in-memory editor state** — the editor screen must not remount the engine/canvas on locale change (story 1.9 acceptance 1: "without a full reload disrupting their work").
- Choice persists (settings store + cookie) and is restored on next launch (FR-004).

## Export-path text

Text overlays rendered into exported images do not go through this catalog (they're user content), but share the rules: NFC normalization and verified-Vietnamese fonts only (see edit-state.md Layer rules, FR-302/FR-402).
